Mastering Variable Assignment in Python 3

A deep dive into the mechanical nature of pointers, memory management, and the philosophy of dynamic typing within the Python ecosystem.

In the architectural framework of Python, a variable is more than a container; it is a label attached to an object. Unlike statically typed languages where a variable is a reserved memory location of a fixed size, Python variables are symbolic names that act as pointers to objects residing in memory.

basic_assignment.py content_copy 
# Assigning an integer
x = 42

# Reassigning to a string (Dynamic Typing)
x = "Aurelian Technical"

# Multiple assignment
a, b, c = 5, 10, 15

The Philosophy of Dynamic Typing

Python’s dynamic nature allows for rapid prototyping and fluid structural changes. This flexibility, however, demands a disciplined approach to naming conventions. PEP 8 suggests using snake_case for variable names, ensuring readability across complex codebases.

check_circle

Valid Names

  • user_id = 101
  • _internal_cache = {}
  • MAX_CONSTANT = 99
cancel

Invalid Names

  • 2nd_user = "Error"
  • user-name = "Error"
  • global = "Keyword"

Memory Address Observation

To truly understand the "label" concept, one must observe the memory address. The id() function reveals the unique identifier of the object the variable is currently pointing to. 

y = 100
print(id(y))  # Output: 14073289...

z = y
print(id(z))  # Same Output: Both point to same object
lightbulb

Aurelian Lab Tip

Always remember that Python optimizes memory for small integers (typically -5 to 256) by pre-allocating them in memory. This is called "Integer Interning."
