# Proposal: ToolkitX 重构 - 运维工具箱

## 问题

当前 ToolkitX 本质上是 SSH 命令的 Web UI 包装，没有独特价值：
- 直接连客户服务器/数据库 → 合规风险、审计问题
- 跳板机断了就无法使用 → 依赖性强
- 跟 xShell/SecureCRT 没有本质区别 → 用户没有迁移动力
- 功能跟网上现有工具一样 → 没有差异化

## 解决方案

重新定位为 **"运维工具箱 + AI 助手"**，不需要权限就能用，每个功能都解决真实痛点。

### 核心理念
- 不需要客户权限（合规无问题）
- 引导式设计（不会也能用）
- AI 辅助（描述需求 → 生成代码）
- 一站式工具箱（不用到处找工具）

## 模块规划

### 模块 1: 配置生成器（引导式 + AI）

| 功能 | 说明 |
|------|------|
| Docker Compose 生成器 | 选服务 → 填参数 → 生成配置，每项有说明 |
| Nginx 配置生成器 | 反向代理/静态站点/负载均衡，附带 SSL 配置 |
| Systemd 服务生成器 | 指定工作目录、用户、环境变量 |
| Cron 表达式生成器 | 可视化选择 + 实时预览下次执行时间 |
| AI 配置生成 | 用户描述需求，AI 生成配置 |

### 模块 2: 脚本库 + AI 生成

| 功能 | 说明 |
|------|------|
| Python 自动化脚本 | 批量巡检、日志分析、自动备份、监控告警 |
| SQL 脚本 | 建表语句生成器（填字段→出SQL）、数据迁移、同步 |
| Shell 脚本 | 批量操作、部署脚本、环境初始化 |
| AI 脚本生成 | 描述需求 → AI 生成完整脚本 + 使用说明 |

### 模块 3: 诊断工具（从服务器视角）

| 功能 | 说明 |
|------|------|
| 端口探测 | 从服务器发起 TCP 连接测试 |
| DNS 查询 | 查询 A/AAAA/MX/CNAME/TXT 记录 |
| SSL 证书检查 | 证书有效期、签发者、域名匹配 |
| HTTP 响应检测 | 状态码 + 响应时间 + Headers 检查 |
| MTR/Traceroute | 路径追踪 + 丢包分析 |
| 诊断报告 | 自动生成排查结论，可导出 |

### 模块 4: 数据处理

| 功能 | 说明 |
|------|------|
| JSON 工具 | 格式化/压缩/校验/JSON↔YAML↔CSV 互转 |
| 编码工具 | Base64 编解码、URL 编解码、JWT 解析 |
| 时间戳工具 | Unix↔人类可读 互转、时区转换 |
| 密码生成器 | 自定义长度、字符集 |
| 哈希计算 | MD5/SHA256/SHA512 |

### 模块 5: 在线终端（运维工作台）

| 功能 | 说明 |
|------|------|
| SSH 连接 | 只连自己的服务器（合规） |
| 快速命令库 | 常用命令分类收藏，一键插入 |
| 命令笔记本 | 执行过的命令自动记录，按项目分组 |
| 会话快照 | 保存终端状态，分享给同事 |
| 终端 + 诊断联动 | 诊断发现问题 → 一键跳终端排查 |

## 合规性

- ✓ 不存储客户服务器信息
- ✓ 不连接客户数据库
- ✓ SSH 只连自己的服务器
- ✓ 诊断工具从自己网络出发
- ✓ 所有操作有审计日志

## 技术栈

- Next.js 16 + React 19 + TypeScript
- Tailwind CSS 4 + shadcn/ui（暗色主题）
- xterm.js（终端）
- recharts（图表）
- CodeMirror（代码编辑）
- ssh2（SSH 连接）
