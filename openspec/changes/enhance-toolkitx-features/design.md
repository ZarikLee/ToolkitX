## Context

ToolkitX 是一个面向运维工程师的 Web 工具平台，目前已实现基础的容器管理、服务器管理和数据库管理功能。当前版本功能较为简单，无法满足实际运维工作需求。需要增强现有功能并新增专业工具模块。

## Goals / Non-Goals

**Goals:**
- 提供 Web SSH 终端连接远程服务器
- 提供远程文件管理器
- 支持 Docker Compose 文件管理
- 实现实时系统指标监控
- 支持进程和服务管理
- 实现 SQL 查询历史和收藏
- 完善设置页面和主题切换

**Non-Goals:**
- 不实现多用户权限系统
- 不实现容器编排（Kubernetes）
- 不实现实时告警通知
- 不实现自动化运维流水线

## Decisions

### SSH 终端实现：xterm.js + WebSocket
- **选择**: xterm.js 作为前端终端，通过 WebSocket 连接后端 SSH
- **理由**: xterm.js 是标准的 Web 终端方案，支持 ANSI 转义序列
- **替代方案**: 终端模拟器（功能有限）、直接 SSH 客户端（安全性差）

### 文件管理器：自研组件
- **选择**: 基于 React 的自研文件管理器
- **理由**: 需要与 SSH 终端集成，支持远程文件操作
- **替代方案**: 第三方文件管理器（集成复杂）

### 系统监控：recharts 图表
- **选择**: recharts 作为图表库
- **理由**: React 原生支持、声明式 API、轻量级
- **替代方案**: Chart.js（命令式 API）、D3.js（学习曲线陡）

### SQL 编辑器：CodeMirror
- **选择**: CodeMirror 6 + @codemirror/lang-sql
- **理由**: 语法高亮、自动补全、SQL 格式化
- **替代方案**: Monaco Editor（体积大）、Ace Editor（维护不活跃）

### 数据存储：JSON 文件
- **选择**: 继续使用 JSON 文件存储配置
- **理由**: 简单、无需额外数据库、适合工具类应用
- **替代方案**: SQLite（增加依赖）

## Risks / Trade-offs

- **[安全性]** Web SSH 终端存在安全风险 → 仅允许本地部署、不暴露到公网、后续添加认证
- **[性能]** 实时监控可能影响响应速度 → 使用 WebSocket 推送、限制刷新频率
- **[兼容性]** 不同操作系统 SSH 配置可能不同 → 支持常见配置、提供默认值
- **[依赖]** 新增依赖增加包体积 → 按需加载、Tree Shaking
